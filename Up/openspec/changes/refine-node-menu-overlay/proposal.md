# Proposal: Refine Nodes Menu Overlay

## Background
- 已有 `Nodes` 子菜单实现为侧边浮层，锚定在左侧导航旁，用户反馈难以注意到，也未能“在右侧弹出子菜单”。
- 当前主内容区域在用户初次点击 `Nodes` 时仍停留在上一次浏览的创建/管理界面，违背了“点击后再选择操作”的预期。
- 需要在保持现有 create/manage 能力的前提下，提供更显著、右侧主区域的操作面板。

## Goals
1. 将 `Nodes` 子菜单迁移到工作区主区域（右侧），突出显示“新建节点”“查看已创建节点”等操作。
2. 引入显式的 `menu` 视图状态，与 `create`、`manage` 同级，确保用户在选择操作前不会看到旧内容。
3. 更新返回/切换逻辑，使“返回节点菜单”按钮回到新的主区域面板而非侧边浮层。
4. 保持节点表单、节点列表现有功能可用，保存后依旧跳转到 manage 视图并刷新数据。
5. 清理过时的浮层定位样式与监听逻辑，避免额外的 DOM 事件与定位 bug。

## Non-Goals
- 不新增节点操作类型或改动后端契约。
- 不改动其他导航标签（Prompts/Workflow/Variables/Logs）的体验。
- 不引入新的权限、动画或国际化方案。

## Deliverables
- 更新 `PipelineWorkspace.vue` 的视图状态管理与布局，移除浮层定位逻辑。
- 更新或扩展 `NodeSubMenu.vue`，适配主区域卡片式布局。
- 根据需要调整相关样式文件，确保主区域菜单展示良好。
- 更新或新增 `docs/ProjectDev` 或相关文档片段，说明新的节点菜单交互。
